第一部分：预训练 (Pre-train)

预训练是 LLM 的基石，主要任务是让模型通过海量无标注文本学习语言知识和世界知识。

1.1 模型与权重加载

预训练模型加载主要有两种方式：从零开始（From Scratch）和基于现有模型继续预训练（Incremental Pre-training）。

核心概念

  • AutoConfig: 用于加载模型的结构配置（层数、隐藏层大小等），不包含权重。
  • AutoModelForCausalLM: 用于加载因果语言模型（Causal Language Model, CLM），即标准的 GPT 结构。

[关键解析] 文本拼接与分块 (Grouping)

为什么要做 Grouping？

普通的 tokenize 会导致每条数据长度不一。如果直接训练，短数据需要填充（Padding）大量无意义的 token，浪费计算资源；长数据会被截断，丢失信息。

解决方案：

将所有 token 串联成一个超长序列，然后按照固定的 block_size（如 2048 或 4096）进行切分。这样每个 batch 的数据都是填满的，极大提高了训练效率。

第二部分：监督微调 (SFT, Supervised Fine-Tuning)

SFT 的目的是让模型学会“遵循指令”。与预训练不同，SFT 是有监督的，数据是成对的 (Instruction, Response)。

2.1 SFT 与 Pre-train 的核心区别

特性 Pre-train (预训练) SFT (监督微调)
数据格式 纯文本 问答对 (Prompt + Response)
学习目标 预测任意位置的下一个 token 根据 Prompt 预测 Response
Loss 计算 所有 Token 都计算 Loss 仅 Response 部分 计算 Loss (关键!)

2.2 SFT 数据处理与 Masking 机制

SFT 数据处理的核心难点在于：如何构造输入序列，并让模型只学习回答部分，而不学习提问部分。

  • <|im_end|>: 句子结束
